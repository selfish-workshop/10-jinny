# 운친 (運親) — PRD

## 1. 서비스 개요
운친은 운동 입문자를 대상으로 한 소셜 동기부여 웹 서비스다. 친구들의 운동 기록이 피드에 올라오는 것만으로 "나도 해야겠다"는 동기를 만드는 것이 핵심 가치다. 운동을 가르치거나 분석하지 않고, 하고 싶게 만드는 데 집중한다.

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

## 3. 기획 (Planning)
- 빠른 운동 기록 (30초 이내: 운동명 + 시간 + 사진 + 한줄코멘트)
- 친구 피드 (SNS형 타임라인, 좋아요/댓글)
- 다중 그룹 시스템 (기록 1회 → 전체 그룹 자동 공유)
- 챌린지 (친구/그룹 단위 기간 목표)
- 스트릭 & 뱃지 동기부여

상세는 `planning/` 폴더 참조. [작성 예정]

## 4. 디자인 (Design)
에너지풍 / 짜릿한 톤&매너. 오렌지·레드 계열 메인 컬러. 5개 핵심 화면: 홈피드 / 기록하기 / 챌린지 / 알림 / 프로필.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
타겟 페르소나: 운동은 알지만 혼자 하면 금방 그만두는 20~30대. 카카오/구글 로그인 → 닉네임 설정 → 첫 기록 유도의 3단계 온보딩.

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)
첫 사용자 확보 전략 및 채널별 전략.

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)
수익 모델 미결정. 사용자 확보 우선, 이후 freemium 전환 고려.

상세는 `strategy/` 폴더 참조. [작성 예정]

## 8. 역할 배정 (가상)

| 팀 | 담당 영역 | 작업 폴더 |
|----|---------|----------|
| PM (본인) | 전체 기획·의사결정 | 루트 |
| 기획 팀 | 기능 정의, MVP 범위 | planning/ |
| 디자인 팀 | UI/UX, 비주얼 | design/ |
| CX 팀 | 페르소나, 여정맵, FAQ | cx/ |
| 마케팅 팀 | 런칭, 채널 | marketing/ |
| 전략 팀 | 수익 모델, 성장 | strategy/ |
